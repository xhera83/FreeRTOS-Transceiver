◆ addDataAllocateCallback()

void FRTT::FRTTransceiver::addDataAllocateCallback ( void(*)(const FRTTDataContainerOnQueue &, FRTTTempDataContainer &)  fP)

To to add an allocator callback function.

This method is used to add an allocator callback function to the library. When the library reads data off the queue, it will do so by copying its content into a FRTTDataContainerOnQueue structure.
After that, the library aquires the next free space in the internal rx buffer (FRTTTempDataContainer) and places a call to the allocator callback
Now the callback function will be called, in which the user of the library can just copy everything (pointer to the original data of communication partner) over OR allocate memory (e.g malloc) and then copy data into own variables.

Parameters
fPPointer to a function which takes FRTTDataContainerOnQueue and FRTTTempDataContainer as an argument.
Returns
void
Attention
Later versions of the library will support some sort of memory pool to allocate/free data through the callbacks

◆ databroadcast()

bool FRTT::FRTTransceiver::databroadcast ( uint8_t  u8Datatype,
void *  data,
int  blockTimeWrite_Ms,
int  blockTimeTakeSemaphore_Ms,
uint32_t  u32AdditionalInfo 
)

Broadcasts data to all communication lines with a tx connection.

Parameters
u8DatatypeDatatype of void pointer 'data'. (Create your own system wide datatypes)
dataPointer to the original data
blockTimeWrite_MsHow long the task calling this method should be put into a blocked state (max) until it is allowed to write to ONE of the queues (milliseconds)
blockTimeTakeSemaphore_MsHow long the task calling this method should be put into a blocked state (max) until is is allowed to aquire ONE of the semaphores for the operation (milliseconds)
u32AdditionalInfo32 Bit additional payload along with void pointer 'data'
Returns
True if it was successfully written to ALL tx connections.
False if at least one write operation failed
Note
Supplied block-time's will be used for each write operation!
Databroadcast will proceed even if a write operation failed
Here you can pass a 32 bit variable along with the data pointer.
You can have the same functionality but with an additional 64 bit payload instead, by uncommenting FRTTRANSCEIVER_64BITADDITIONALDATA in FRTTransceiverSettings.h
Attention
Do not use this feature when you have a lot of tx connections and it's particularly hard to get instant permission to write on at least one of them
However, if you still need to, at least keep blockTimeWrite_Ms and blockTimeTakeSemaphore_Ms very low
